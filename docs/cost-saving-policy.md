# 비용 절감 정책 (LLM·API)

## 원칙

1. **짧고 고정된 컨텍스트**만 모델에 넘긴다. 전체 기획서 원문을 매 요청에 붙이지 않는다.
2. **섹션 단위**로 호출한다. 한 번에 “전 페이지”를 요청하지 않는다.
3. **캐시**: 동일 입력(프롬프트+파라미터)에 대한 결과는 재사용 가능하면 재사용한다.
4. **모델 선택**: 초안·짧은 수정은 가벼운 모델, 최종 톤 다듬기만 상위 모델 등 단계 분리를 검토한다.
5. **프롬프트에 출력 형식 명시**(JSON/Markdown 등)해 재시도와 파싱 비용을 줄인다.

## 구현 체크리스트

- [ ] 사용자 입력을 “요약 슬롯”으로 정규화한 뒤 모델에 전달
- [ ] 섹션별 system/user 프롬프트 분리 (`prompts/` 참고)
- [ ] “수정” 요청 시 **이전 섹션 텍스트만** 첨부 (전체 페이지 X)
- [ ] 실패 시 지수 백오프·에러 로그에 본문 전체 저장 금지

## 로깅

- 프롬프트/응답 전문은 기본 저장하지 않는다. 필요 시 마스킹·짧은 스니펫만.
